# Architecture

## Frontend

- `apps/web` — Next.js (web application)
- `apps/mobile` — Expo React Native (mobile application)

---

## Shared Packages

These packages are shared across the monorepo to avoid code duplication between web, mobile, and backend.

- `@repo/auth` — shared authentication utilities
- `@repo/api` — frontend API client layer
- `@repo/core` — shared business logic and types
- `@repo/config` — shared configuration
- `@repo/ui` — shared UI components
- `@repo/db` — database schema and backend documentation

---

## Backend Infrastructure

The backend is serverless and hosted on AWS.

- **Amazon Cognito** — authentication (signup, login, JWT tokens)
- **API Gateway** — HTTP API routing
- **AWS Lambda** — backend application logic
- **PostgreSQL (Amazon RDS)** — primary relational database
- **PostGIS** — geospatial queries for location-based discovery
- **Amazon S3** — media storage

S3 is used for:

- profile images
- post media
- chat media
- stories

Uploads are performed using **presigned URLs generated by the backend**.

---

## Backend Domains

### Users
Handles user profiles and preferences.

Data stored includes:

- name
- age
- bio
- gender
- sexual orientation
- interests
- introversion score

Users are linked to **Cognito accounts** via the `cognito_sub`.

---

### Location

Stores user geographic coordinates.

Used for:

- nearby-user discovery
- distance calculation

Location queries use **PostGIS geographic points**.

---

### Discovery

Nearby discovery is based on:

1. **PostGIS radius filtering**
2. **Mutual compatibility filtering**
3. **Compatibility ranking algorithm**

Ranking uses:

- shared interests
- geographic distance
- introversion similarity

Final ranking score:
final_score =
0.50 × interest_score +
0.30 × distance_score +
0.20 × introversion_similarity

---

### Matching

Users can like other users.

If two users like each other:

- a **match is created**
- chat becomes available between them

Rate limit: **100 likes per 24 hours**.

---

### Chat

Chat functionality includes:

- message sending
- message history
- pagination
- read receipts
- typing indicators
- message editing
- message soft deletion

Messages are stored in the `messages` table.

---

### Posts

Users can create posts containing:

- text
- media
- or both

Features:

- post likes
- comments
- feed retrieval
- optional expiration timestamps

---

### Stories

Stories work similarly to Instagram stories.

Features:

- image or video media
- captions
- view tracking
- automatic expiration after **24 hours**

Story views are stored in the `story_views` table.

---

### Media Upload

Media uploads are handled through **S3 presigned URLs**.

Supported folders:

- `profile-images`
- `posts`
- `chat-media`

Supported file types:

- JPEG
- PNG
- WEBP
- GIF
- MP4
